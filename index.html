Pushing to a GitHub Repository

If you are creating a new local Git repository...
1. git init -b main to initialize the local directory as a Git repository.
2. git remote add origin 'url' to add a url to the repository to connect on GitHub.
If you are trying to clone an existing repository...
1. git clone 'url' to copy the repository on the local directory.
2. git pull origin 'branch name' in order to update the local repository with the newest version of GitHub repository

Now that your local Git repository is ready, it is time to push your work to the GitHub repository.
1. git add . to add files to your Git local repository.
2. git commit -m 'message' to commit the files that you have staged in your local repository.
3. git push origin 'main or another branch' to push the local repository to the GitHub repository.

To reset the link to the remote repository
1. git remote set-url origin 'new-url'
To clone a specific branch
1. clone -b 'branch-name' 'repository url'
To create a new branch for development
1. git checkout -b 'branch-name'
To check whether a branch has been created
1. git branch
